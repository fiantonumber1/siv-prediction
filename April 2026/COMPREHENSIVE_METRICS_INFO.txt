## 📊 IMPLEMENTED METRICS

### 1️⃣ FORECASTING METRICS (Signal Prediction)

#### MSE (Mean Squared Error)
- **What**: Average squared difference between predicted and actual values
- **Why**: Penalizes large errors more heavily
- **Interpretation**: Lower is better, sensitive to outliers
- **Scale**: Same as data squared

#### RMSE (Root Mean Squared Error)
- **What**: Square root of MSE
- **Why**: Same scale as original data, more interpretable
- **Interpretation**: Lower is better, represents average error magnitude
- **Scale**: Same as data

#### MAE (Mean Absolute Error)
- **What**: Average absolute difference between predicted and actual
- **Why**: More robust to outliers than MSE, easier to interpret
- **Interpretation**: Lower is better, average error magnitude
- **Scale**: Same as data

#### MAPE (Mean Absolute Percentage Error)
- **What**: Average percentage error
- **Why**: Scale-independent, relative error measure
- **Interpretation**: Lower is better, <10% is excellent, 10-20% is good
- **Scale**: Percentage (%)

#### R² Score (Coefficient of Determination)
- **What**: Proportion of variance explained by the model
- **Why**: Measures goodness of fit, scale-independent
- **Interpretation**: 0-1 (higher is better), 1 = perfect fit, 0 = baseline
- **Scale**: 0 to 1

#### Avg MSE Per Feature
- **What**: Average MSE calculated separately for each of 21 parameters
- **Why**: Identifies which parameters are harder to predict
- **Interpretation**: Lower is better, shows per-parameter performance
- **Scale**: Same as data squared

### 2️⃣ CLASSIFICATION METRICS (Health Status)

#### Accuracy
- **What**: Percentage of correctly classified samples
- **Why**: Overall performance measure
- **Interpretation**: Higher is better, >90% is excellent
- **Limitation**: Can be misleading with imbalanced classes

#### Precision (Macro Average)
- **What**: Average precision across all classes (unweighted)
- **Why**: Measures how many predicted positives are correct
- **Interpretation**: Higher is better, important when false positives are costly
- **Formula**: TP / (TP + FP)

#### Recall (Macro Average)
- **What**: Average recall across all classes (unweighted)
- **Why**: Measures how many actual positives are detected
- **Interpretation**: Higher is better, important when false negatives are costly
- **Formula**: TP / (TP + FN)

#### F1-Score (Macro Average)
- **What**: Harmonic mean of Precision and Recall
- **Why**: Balances precision and recall
- **Interpretation**: Higher is better, >0.8 is good
- **Formula**: 2 × (Precision × Recall) / (Precision + Recall)

#### Precision/Recall/F1 (Weighted Average)
- **What**: Averages weighted by class support
- **Why**: Accounts for class imbalance
- **Interpretation**: Better reflects real-world performance

#### Per-Class Metrics
- **What**: Precision, Recall, F1 for each class separately
- **Why**: Identifies which classes the model struggles with
- **Classes**: 
  - Class 0: Sehat (Healthy)
  - Class 1: Pre-Anomali (Pre-Anomaly)
  - Class 2: Near-Fail (Critical)
- **Interpretation**: Helps identify model weaknesses

#### Confusion Matrix
- **What**: Matrix showing actual vs predicted classifications
- **Why**: Detailed view of misclassification patterns
- **Format**: Rows = Actual, Columns = Predicted
- **Interpretation**: Diagonal = correct predictions

#### ROC-AUC (One-vs-Rest)
- **What**: Area under ROC curve for multi-class (OvR strategy)
- **Why**: Measures discriminative ability across all classes
- **Interpretation**: 0.5-1.0, >0.9 is excellent, <0.7 needs improvement
- **Scale**: 0 to 1

## 📈 WHEN METRICS ARE CALCULATED

### During Training:
- **Every Epoch**: Basic metrics (Loss, MSE, Accuracy) for both train and validation
- **Every 10 Epochs**: Comprehensive metrics report (all metrics above)
- **Logged To**: `training_log_21param_TCN.txt`

### After Training:
- **Full Dataset Evaluation**: All metrics calculated on entire dataset
- **Classification Report**: Saved to `classification_report_final.txt`
- **Confusion Matrix**: Logged and saved with detailed format

---

## 🎯 HOW TO USE THESE METRICS

### Model Performance Assessment:
1. **Check R² Score**: Should be >0.8 for good fit
2. **Check MAPE**: Should be <20% for reliable predictions
3. **Check F1-Score**: Should be >0.8 for all classes
4. **Check ROC-AUC**: Should be >0.9 for excellent discrimination

### Identifying Weaknesses:
1. **Per-Class Metrics**: Low recall on "Near-Fail" = missing critical failures
2. **Per-Feature MSE**: High MSE on specific parameters = need more training data
3. **Confusion Matrix**: Off-diagonal patterns = systematic misclassification

### Model Comparison:
1. **Validation Loss**: Primary metric for early stopping
2. **Val F1-Score**: Compare classification performance
3. **Val MAPE**: Compare forecasting accuracy
4. **Val ROC-AUC**: Compare discriminative ability

## 🔧 IMPLEMENTATION DETAILS

### Data Collection:
- Predictions and ground truth collected during training/validation loops
- Classification probabilities saved for ROC-AUC calculation
- Signal predictions saved for forecasting metrics

### Metric Calculation:
- **Scikit-learn** functions used for reliability
- **Multi-class** metrics use macro/weighted averaging
- **ROC-AUC** uses One-vs-Rest (OvR) strategy
- **Zero division** handled gracefully (zero_division=0)

### Reporting:
- Metrics formatted with emojis for readability
- Reports saved to text files for permanent record
- Confusion matrix displayed with class labels
- Per-class breakdowns enable detailed analysis
